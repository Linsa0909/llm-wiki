# 一、核心概念

## 导入来源：二进制协议解析指南.md

## 一、核心概念

### 1.1 字节序 (Endianness)

```
大端 (Big-Endian, BE, 网络序):
  高位字节存低地址
  0x12345678 → 内存: [12][34][56][78]
  使用场景: 网络协议(TCP/IP), Modbus, CAN

小端 (Little-Endian, LE, Intel序):
  低位字节存低地址
  0x12345678 → 内存: [78][56][34][12]
  使用场景: x86 CPU, 大多数 ARM MCU, 很多嵌入式设备

判断方法:
  读到一个 2 字节值 0x0100:
    如果原始字节是 [01][00] → 大端 → 值是 256
    如果原始字节是 [00][01] → 小端 → 值是 256
```

### 1.2 实践：从一串十六进制字节还原数值

```
原始字节: AA 55 0C 00 00 00 64 00 00 80 3F C0

逐字段解析（假设协议: 帧头2B + 长度4B + 值4B + 校验4B）:

  AA 55          → 帧头 = 0xAA55 (大端)
  0C 00 00 00    → 长度 = 0x0C = 12 (小端)
  64 00 00 80    → 手动: 0x80000064 = -2147483548 (小端, 有符号)
  3F C0 ...      → 开始校验字段
```
