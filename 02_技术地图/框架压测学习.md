#### 2.1 python正则表达式

正则表达式（Regular expressions，也叫 REs、regexs 或 regex patterns）

涉及模块：import re

涉及代码：

```
match_begin = 
re.search
(r"\[(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})\.(\d{3})\.(\d{3}).*?\] register service \[id:(\d+) name:(.*?)\] begin!", 
line_str)
```

匹配内容：

```
[2026-03-16 10:10:04.505.145 WARN ser2.cpp:start:51 zj1@288722944@3618702 127.0.0.1 3618702] register service [id:1397 name:ser2] begin!
```



`r` 是 Raw String（原始字符串）的缩写。它的作用只有一个——“剥夺 Python 解释器对转义字符的处理权，把字符串原封不动地交给正则表达式引擎。即看到了r ，字符串里面的反斜杠都不碰！



\ [ , 表示匹配字符 [ , 因为在正则表达式中 [ ] 表示一个字符集合 。

比如 `[abc]`，它的意思是：“这里只要出现 a、b、c 中的**任意一个**，就算匹配成功”。



. * ? 解释

1. **`.` (点号)**：正则里的万能牌，代表**“任意一个字符”**（除了换行符）。
2. **`*` (星号)**：代表前面那个字符可以出现**“0 次或无数次”**。所以 `.*` 加在一起的意思就是：**吃掉所有字符，有多少吃多少。** 这在正则里叫**“贪婪模式 (Greedy)”**。
3. **`?` (问号) —— 绝对的核心！**：它跟在 `*` 后面，意思是**“非贪婪模式 (Lazy/Reluctant)”**。它告诉引擎：“吃得越少越好，只要满足后面的条件，立刻停嘴！”





#### 2.2 读取stdout

涉及代码：

```
while True:
    line = await process.stdout.readline()
    if not line:
        break

    line_str = line.decode('utf-8', errors='ignore').strip()
    line_lower = line_str.lower()
```

readline（） 极其严格的按行读取，而且底层性能最高效的读法。

在计算机操作系统的底层，没有“行”这个概念，只有字节流 Bytes。

隐形字符：`\n`（换行符）。吞进内存缓冲区 ，直至遇到  \ n 然后截断复制给line



.strip() 去空格



await：  等待 ，等到 读完一行 然后马上回来继续执行  异步协程 (Asyncio)
