# V4L2 摄像头验证命令

## 枚举多个 video 节点

当设备枚举出多个 `/dev/videoX` 时，需要分别确认哪个是 Video Capture，哪个可能只是 Metadata Capture。

```bash
for d in /dev/video59 /dev/video60; do
    echo "===== $d ====="
    v4l2-ctl -D -d "$d"
    v4l2-ctl --list-formats-ext -d "$d"
done
```

## 命令拆解

| 命令 | 作用 |
|---|---|
| `v4l2-ctl -D -d "$d"` | 查看设备驱动、Card、Bus、Capability 等信息 |
| `v4l2-ctl --list-formats-ext -d "$d"` | 查看该设备支持的像素格式、分辨率和帧率 |

## 指定格式做流测试

```bash
v4l2-ctl -d /dev/video59 \
  --set-fmt-video=width=2080,height=1080,pixelformat=MJPG \
  --set-parm=60 \
  --stream-mmap=4 \
  --stream-count=300 \
  --stream-to=/dev/null \
  --verbose
```

## 参数说明

| 参数 | 含义 |
|---|---|
| `-d /dev/video59` | 指定摄像头设备节点 |
| `--set-fmt-video=width=2080,height=1080,pixelformat=MJPG` | 设置采集格式为 2080x1080 MJPG |
| `--set-parm=60` | 设置目标帧率为 60fps |
| `--stream-mmap=4` | 使用 mmap 方式申请 4 个缓冲区进行流测试 |
| `--stream-count=300` | 采集 300 帧后停止 |
| `--stream-to=/dev/null` | 丢弃输出，只验证采集链路是否稳定 |
| `--verbose` | 输出更详细的 ioctl/格式协商信息 |

## 判断结果

如果命令能稳定采完 300 帧，说明该设备节点在指定格式和帧率下能正常出流。如果报格式不支持、帧率设置失败或 ioctl 错误，应回到 `--list-formats-ext` 查看设备真实能力。
