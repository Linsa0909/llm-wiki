### 7.1 肉眼解析十六进制

```
技巧: 把字节序列打印成格式化的十六进制字符串

推荐格式:
  0000: AA 55 01 02 0C 00 00 00  80 3F 00 00 00 40 CD CC  |.U.......?...@..|
  0010: 4C 3F 00 00 40 60 B2 3F  1A 2B                    |L?..@`.?.+     |

Python:
  import binascii
  for i in range(0, len(data), 16):
      chunk = data[i:i+16]
      hex_str = ' '.join(f'{b:02X}' for b in chunk)
      ascii_str = ''.join(chr(b) if 32 <= b < 127 else '.' for b in chunk)
      print(f'{i:04X}: {hex_str:<48} |{ascii_str}|')
```

### 7.2 常见 bug 速查

| 症状 | 原因 | 检查方向 |
|------|------|----------|
| 解析的值时对时错 | 字节序搞反 | 交换 `>` 和 `<` |
| float 值离谱 | 大小端不对 / 不是 IEEE 754 | 用已知值验证 |
| 帧一直解析失败 | 帧头不匹配/长度字段位置不对 | 打印原始字节对比协议文档 |
| 偶尔丢帧 | CRC 错误 / 缓冲区溢出 | 计算 CRC 在发送端和接收端对比 |
| 解析到一半就断了 | 粘包/拆包处理不完整 | 检查状态机逻辑 |
| 数值整体偏移 | 单位转换系数漏了 | 确认协议中的 scale/offset |
| 快速发送时丢数据 | 串口缓冲区满 | 增大缓冲区 / 加流控 |

### 7.3 一套完整的调试打印

```python
def debug_frame(data: bytes, label: str = "Frame"):
    """格式化的帧调试打印"""
    print(f"\n{'='*60}")
    print(f"  {label} ({len(data)} bytes)")
    print(f"{'='*60}")

    # 十六进制 dump
    for i in range(0, len(data), 16):
        chunk = data[i:i+16]
        hex_str = ' '.join(f'{b:02X}' for b in chunk)
        print(f"  {i:04X}: {hex_str}")

    # 自动推断常见字段
    if len(data) >= 2:
        header_be = struct.unpack('>H', data[:2])[0]
        header_le = struct.unpack('<H', data[:2])[0]
        print(f"\n  帧头 (BE): 0x{header_be:04X}")
        print(f"  帧头 (LE): 0x{header_le:04X}")

    if len(data) >= 4:
        print(f"  Bytes 2-3 (BE): {struct.unpack('>H', data[2:4])[0]}")
        print(f"  Bytes 2-3 (LE): {struct.unpack('<H', data[2:4])[0]}")

    print(f"{'='*60}\n")
```
