# Mino17 RK3588 红外采集推流复盘

## 一句话结论

本项目已经把原先基于 `ffmpeg` 命令行的采集推流流程，改造成 **C++ 直接调用 FFmpeg/libav 库 API** 的方式，并在 RK3588 上通过 `h264_rkmpp` 完成 H264 硬件编码，再推流到 ZLMediaKit 分发。

## 我这次真正学到的宏观概念

- Linux 摄像头不是“软件对象”，而是 `/dev/videoX` 这类设备文件。
- `v4l2-ctl --list-devices` 可以查看板子识别到的摄像头设备。
- Mino17 这次识别为 `/dev/video59`，另一个 `/dev/video60` 是 metadata capture，不是主画面。
- C++ 并不是调用 `ffmpeg` 命令，而是调用 FFmpeg 的库：`libavdevice`、`libavformat`、`libavcodec`、`libswscale` 等。
- RK3588 的硬件编码通过 `h264_rkmpp` 使用 MPP 能力。
- ZLMediaKit 接收 RTMP 后，可以同时分发 RTMP、HTTP-FLV、HLS、RTSP。
- VLC/ffplay 只是播放器，用来拉流验证画面是否正常。

## 最终链路

```text
Mino17 摄像头
  -> /dev/video59
  -> C++ 程序通过 FFmpeg/libav API 采集 V4L2
  -> 裁剪 640x518 到 640x512
  -> 伪彩和格式转换
  -> h264_rkmpp 硬件编码
  -> RTMP 推流
  -> ZLMediaKit 接收
  -> VLC/ffplay/Web/NVR 播放
```

## 是否满足需求

满足。

原目标是：

> 要把 ffmpeg 的命令调用，改为通过 C/C++ 调用库的方式。

当前正式采集链路中没有执行 `ffmpeg` 命令。C++ 程序直接调用：

```cpp
av_find_input_format("v4l2");
avformat_open_input(...);
avcodec_send_packet(...);
avcodec_receive_frame(...);
sws_scale(...);
avcodec_find_encoder_by_name("h264_rkmpp");
av_interleaved_write_frame(...);
```

## 当前项目结果

- 输入：`640x518 UYVY 50fps`
- 输出：`640x512 H264 50fps`
- 编码：`h264_rkmpp`
- 码率：约 `500 kbps`
- 推流：`rtmp://127.0.0.1:1935/ghostapp/uav0`
- 播放：`ffplay rtmp://<板子IP>:1935/ghostapp/uav0`
- 部署包：`Mino17.tar.gz`

## 遇到的问题与解决

| 问题 | 原因 | 解决 |
|---|---|---|
| C++ 编译 `goto cleanup` 报错 | C++ 不能跳过局部变量初始化 | 将变量声明提前，避免 goto 跨初始化 |
| 找不到 `libavcodec.pc` 等 | pkg-config 路径不对或库不完整 | 固定 `PKG_CONFIG_PATH` 指向 `ffmpeg-rockchip-bin/lib/pkgconfig` |
| 找不到 `libavdevice` / v4l2 input format | FFmpeg 构建或链接没有带 avdevice | Makefile 增加 `libavdevice`，Dockerfile 确保依赖存在 |
| 链接缺 `librga`、`libswresample`、`libx264` | FFmpeg/RGA 运行依赖没有显式链接或安装 | 补齐 Makefile、Dockerfile、apt manifest |
| GLIBC 版本不兼容 | 板子系统 glibc 较低，外部库构建环境不匹配 | 改为在 Ubuntu 22.04 ARM64 Docker 内构建/运行 |
| 设备路径不一致 | 新板子枚举为 `/dev/video59` | 用 `v4l2-ctl --list-devices` 和 `CAMERA_DEVICE=/dev/video59` 指定 |
| ZLM API 查不到 | 端口映射和 secret 不一致 | 固定 ZLM config，映射 `8090:80`，固定 secret |
| Shell 脚本 Linux 执行失败 | Windows 写入 BOM/CRLF | 所有 `.sh` 改为 UTF-8 无 BOM + LF |
| `view.sh` 不友好 | 默认只给 `127.0.0.1`，无说明 | 自动识别板子 IP，增加中文说明、协议区别和后续操作 |
| `status.sh` 输出太多 | 直接输出完整 JSON 和日志 | 默认中文摘要，`--raw`/`--logs` 才输出详细信息 |

## 关键命令沉淀

识别摄像头：

```bash
v4l2-ctl --list-devices
v4l2-ctl -d /dev/video59 --all
```

部署：

```bash
tar -xzf Mino17.tar.gz
cd Mino17_release
chmod +x *.sh scripts/*.sh docker/*.sh
./install.sh
```

看状态：

```bash
./status.sh
./status.sh --raw
./status.sh --logs
```

看画面：

```bash
./view.sh
VIEW_HOST=172.16.3.80 ./view.sh
ffplay rtmp://172.16.3.80:1935/ghostapp/uav0
```
